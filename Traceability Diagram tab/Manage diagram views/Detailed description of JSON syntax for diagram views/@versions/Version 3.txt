<p>A diagram view can be described by a file in a specific JSON format.&nbsp;</p><p>To be able to design diagram views with JSON, you need to know the following information.</p><h2>JSON syntax</h2><p>The following pieces of JSON code are examples of name-value pairs that you can use to <ac:link><ri:page ri:content-title="Manage diagram views" /><ac:plain-text-link-body><![CDATA[create a diagram view]]></ac:plain-text-link-body></ac:link>.</p><h3>Diagram section</h3><p>These settings determine general display settings for the diagram.</p><pre>&quot;visitStrategy&quot;: &quot;directed&quot;,&nbsp;&quot;completeGraph&quot;</pre><ul><li>Determines which nodes and edges have to be traversed and displayed.</li><li>This setting is optional.</li><li>You must pick one of the following values:&nbsp;<ul><li><p><code>&quot;directed&quot;</code>: For the start node, DGC traverses the relations in all directions, and adds all nodes it found. For the nodes it just found, DGC traverses relations only in the same direction as how it got found: if a node was found by traversing an outgoing relation, DGC looks for outgoing relations, and vice versa: if the node was found by traversing an incoming relation, DGC looks only for incoming relations. &nbsp;This behaviour is similar to traversing a hierarchy.&nbsp;<br />This is the default setting.&nbsp;</p></li><li><code>&quot;completeGraph&quot;</code>: All nodes and edges related to the current asset are displayed.<br />&nbsp;</li></ul></li></ul><pre>&quot;resultNodeUnicityStrategy&quot;: &quot;MultipleNodesPerTermId&quot;, &quot;singleNodePerTermId&quot;</pre><ul><li>Determines the number of times a node is displayed if it is found multiple times in the flow of the diagram.</li><li><span style="line-height: 1.4285715;">This setting is optional.&nbsp;</span></li><li><span>You must pick one of the following values:&nbsp;&nbsp;</span><ul><li><p><code>&quot;MultipleNodesPerTermId&quot;</code>: Displays the result node multiple times. This is the default setting.</p></li><li><p><code>&quot;singleNodePerTermId&quot;</code>: Displays the result node only once. <br />NOTE: If an asset is found more than once for the same viewnode, the result is always displayed only once. In that case, there is a loop for that node.<br /><br /></p></li></ul></li></ul><pre>&quot;showEdgeLabels&quot;: false,&nbsp;true</pre><ul><li><p>Determines if the edge labels are displayed.</p></li><li>This setting is optional.</li><li>You must pick one of the following values:&nbsp;<ul><li><p><code>false</code>: The labels are not displayed.&nbsp;This is the default setting.</p></li><li><code>true</code>: The labels are displayed.<br />&nbsp;</li></ul></li></ul><pre>&quot;showLegend&quot;: false,&nbsp;true</pre><ul><li><p>Determines if the diagram legend is displayed. The legend shows all asset types and complex relation types that occur in the diagram.&nbsp;</p></li><li>This setting is optional.</li><li>You must pick one of the following values:&nbsp;<ul><li><p><code>false</code>: The legend is not displayed.&nbsp;</p></li><li><code>true</code>: The legend is displayed.&nbsp;This is the default setting.</li></ul></li></ul><pre>&quot;layout&quot;: &quot;HierarchyLeftRight&quot;, &quot;HierarchyTopBottom&quot;, &quot;Circular&quot;, &quot;SmartOrganic&quot;, &quot;Radial&quot;, &quot;Flow/Context&quot;</pre><ul><li><p><code> <ac:structured-macro ac:name="anchor"><ac:parameter ac:name="">layout</ac:parameter></ac:structured-macro></code>Determines the layout style of the diagram.</p></li><li>This setting is optional.</li><li>You must pick one of the following values:&nbsp;<ul><li><p><code style="line-height: 1.4285715;">&quot;HierarchyLeftRight&quot;</code> <span style="line-height: 1.4285715;">: Nodes and edges are displayed in a flow from left to right. This is the default setting.</span></p></li><li><p><code>&quot;HierarchyTopBottom&quot;</code>: Nodes and edges are displayed in a flow from top to bottom.</p></li><li><code>&quot;Circular&quot;</code>: <span style="color: rgb(64,64,64);">Nodes and edges are&nbsp;</span> <span style="color: rgb(64,64,64);">arranged in a radial tree.</span></li><li><p><code>&quot;SmartOrganic&quot;</code>: <span style="color: rgb(64,64,64);">Nodes and edges are distributed in a well-balanced manner, there are few edge crossings.</span></p></li><li><p><code>&quot;Radial&quot;</code>: <span style="color: rgb(64,64,64);">Nodes and edges are displayed with no overlaps, few edge crossings and few bends.</span></p></li><li><p><code style="line-height: 1.4285715;">&quot;Flow/Context&quot;</code>: Layout style for diagrams with a flow and a context region. See also layoutRegion in the nodes section.&nbsp;<br />Nodes and edges in the flow region are displayed mostly from left to right. The nodes and edges in the context region are displayed above the flow region. <br />An edge that begins or ends with a context node, is shown with less emphasis (thinner and in light grey)&nbsp;than an edge between two flow nodes.<br />If you specify this layout, keep in mind that for an edge between a flow node and a context node, the <code style="line-height: 1.4285715;">&quot;from&quot;</code>&nbsp;node has to be in the flow region and the <code style="line-height: 1.4285715;">&quot;to&quot;</code>&nbsp;node has to be in the context region.</p></li></ul></li></ul><p>&nbsp;</p><pre>&quot;<span style="color: rgb(51,51,51);">maxNodeLabelLength</span>&quot;: 0, &lt;positive integer number&gt;</pre><ul><li>Determines the length of the node labels (whether they should be truncated when they are too long).&nbsp;</li><li><span>This setting is optional.</span></li><li><span>You must provide 0 or a positive integer number as the value:&nbsp;</span><ul><li><code>0</code>: Node labels are not truncated, they are displayed in full length.</li><li><span style="line-height: 1.4285715;">The default setting is 50.&nbsp;</span><br />&nbsp;</li></ul></li></ul><pre>&quot;<span style="color: rgb(51,51,51);">maxEdgeLabelLength</span>&quot;: 0, &lt;positive integer number&gt;</pre><ul><li><p>Determines the length of the edge labels if they have to be truncated when they are too long.&nbsp;</p></li><li>This setting is optional.</li><li>You must provide 0 or a positive integer number as the value:&nbsp;<ul><li><code>0</code>: Edge labels are not truncated, they are displayed in full length.</li><li><span style="line-height: 1.4285715;">The default value is 30.&nbsp;</span></li></ul></li></ul><h3><span style="color: rgb(51,51,51);">Nodes section</span></h3><p><span style="color: rgb(51,51,51);">These settings determine the display settings for nodes. You must add a node for each asset type and complex relation type that you want to include in the diagram.&nbsp;</span></p><pre>&quot;id&quot;: &quot;Business Term1&quot;<code> </code>
</pre><ul><li>Determines the name of the node.</li><li>This setting is mandatory.</li><li>You can type any string here, but it must be unique in this view. For readability, we recommend you use the name of the asset type or complex relation type as the ID.&nbsp;</li><li>You can refer to this node by using its ID in the&nbsp;<code>&quot;from&quot;</code> and <code>&quot;to&quot;</code>&nbsp;name/value pairs of the edges section.<br />&nbsp;</li></ul><pre>&quot;conceptTypeId&quot;: &quot;00000000-0000-0000-0000-000000011001&quot;</pre><ul><li>Determines the resource ID of the asset type or the complex relation type.&nbsp;</li></ul><ul><li>This setting is mandatory.&nbsp;</li><li>You can have multiple nodes with the same conceptTypeId in one diagram view.&nbsp;</li><li>The value has to be a valid resource ID.&nbsp;</li><li>You can find the resource ID of an asset type by clicking <strong style="line-height: 1.4285715;">Settings &gt; Types &gt; Asset types</strong> and then selecting a type. For a node that represents a complex relation type, the label is used in the result diagram as well.&nbsp;</span></li><li><span style="color: rgb(51,51,51);">This setting is optional.&nbsp;</span></li><li><span style="color: rgb(51,51,51);">You can provide any string as the value:&nbsp;</span></li><li><span style="color: rgb(51,51,51);">I<span style="color: rgb(51,51,51);">f you do not specify a label, the name of the asset type or the complex relation type is used.<br />&nbsp;</span> </span></li></ul><pre>&quot;layoutRegion&quot;: &quot;flow&quot;, &quot;context&quot;</pre><ul><li>Determines if the node is treated as a flow node or a context node. This influences the traversal strategy. An edge from a flow node to a context node is always included in the result diagram. <br />An edge between two flow nodes FN1 and FN2 is only included in the diagram if the edge from FN1 to FN2 is traversed in the same direction as the edge that brought FN1 into the diagram. &nbsp;</li><li>This setting is optional.<span style="line-height: 1.4285715;">&nbsp;</span></li><li><span style="line-height: 1.4285715;">You must pick one of the following values:&nbsp;</span><br /><ul><li><code>&quot;flow&quot;</code>: The node is part of the flow. This is the default setting.</li><li><span style="color: rgb(61,60,64);"> <code>&quot;context</code>&quot;: The node is part of the context.&nbsp;</span></li><li><span style="color: rgb(61,60,64);">When you select the &quot;<code>Flow/Context&quot;&nbsp;</code>layout for a diagram, the edges between flow nodes are rendered horizontally, mostly from left to right (the &quot;from&quot; node is to the left of the &quot;to&quot; node). <br />The edges from flow to context nodes are rendered vertically, from bottom to top.</span></li></ul></li></ul><h3><span style="color: rgb(51,51,51);">Edges section</span></h3><p><span style="color: rgb(51,51,51);">These settings determine which directed relations should be traversed, and how they should be depicted on the diagram. Each edge represents a relation type between two nodes (asset types or complex relation types). <br />You have to ensure that the diagram view is a <em>connected</em>&nbsp;graph: each node in the diagram view is reachable from any other node.&nbsp;</span></p><pre>&quot;from&quot;: &quot;Business Term1&quot;</pre><ul><li><span style="color: rgb(61,60,64);">Determines which node is the head asset.</span></li><li>This setting is mandatory.</li><li>You have to fill in a node ID (&quot;id&quot;) from the nodes section.<br />&nbsp;</li></ul><pre>&quot;to&quot;: &quot;Table Column1&quot;</pre><ul><li><span style="color: rgb(61,60,64);"><span style="color: rgb(61,60,64);">Determines which node is the tail asset.</span> </span></li><li>This setting is mandatory.</li><li><span style="color: rgb(61,60,64);">You have to fill in a node ID (&quot;id&quot;) from the nodes section.<br />&nbsp;</span></li></ul><pre>&quot;binaryFactTypeId&quot;: &quot;00000000-0000-0000-0000-000000007038&quot;</pre><ul><li><span style="color: rgb(61,60,64);">Determines the relation type.</span></li><li>This setting is mandatory.</li><li><span style="color: rgb(61,60,64);">You have to fill in the resource ID of the relevant relation type. <br />You can copy and paste these resource IDs from the Settings UI:</span><ul><li><span style="color: rgb(61,60,64);">You can find the resource ID of a relation type by clicking <strong>Settings &gt; Attributes &gt; Relations</strong> and then clicking&nbsp;<ac:image><ri:attachment ri:filename="preview.png" /></ac:image> <strong>&nbsp;&gt; Columns &gt; Resource Id</strong>.</span></li><li><span style="color: rgb(61,60,64);">You can find the resource ID of a complex relation type by clicking&nbsp;<strong>Settings &gt; Attributes &gt; Complex relation types&nbsp;</strong>and then clicking&nbsp;<strong> <ac:image><ri:attachment ri:filename="preview.png" /></ac:image>&nbsp;&gt; Columns &gt; Resource Id</strong>.</span></li><li><span style="color: rgb(61,60,64);">You can find the resource ID of all relation types that are part of a complex relation type by clicking&nbsp;<strong>Settings &gt; Attributes &gt; Complex relation types</strong> &nbsp;and then clicking&nbsp; <ac:image><ri:attachment ri:filename="preview.png" /></ac:image> <strong>&nbsp;&gt; Columns &gt; Relation Ids</strong>.</span> <span style="line-height: 1.4285715;">&nbsp;</span> <br /> <span style="color: rgb(61,60,64);"> <br /> </span></li></ul></li></ul><pre>&quot;roleDirection&quot;: true, false</pre><ul><li><span style="color: rgb(61,60,64);">Determines the direction of the edge; is it the role or co-role.</span></li><li>This setting is mandatory.<span style="color: rgb(61,60,64);">&nbsp;</span></li><li><span style="color: rgb(61,60,64);">You must pick one of the following values:&nbsp;&nbsp;</span><ul><li><span style="color: rgb(51,51,51);"><code><span style="color: rgb(61,60,64);">true</span></code><span style="color: rgb(61,60,64);">: The edge is traversed from head to tail.</span></span></li><li><span style="color: rgb(51,51,51);"><code><span style="color: rgb(61,60,64);">false</span></code><span style="color: rgb(61,60,64);">: The edge is traversed from tail to head.<br />&nbsp;</span></span></li></ul></li></ul><pre>&quot;style&quot;: &quot;arrow&quot;,&nbsp;&quot;box&quot;, &quot;reversed-box&quot;</pre><ul><li><span style="color: rgb(51,51,51);"><span style="color: rgb(61,60,64);">Determines how edges are displayed.</span></span></li><li>This setting is optional.<span style="color: rgb(51,51,51);">&nbsp;<span style="color: rgb(61,60,64);"><br /></span></span></li><li><span style="color: rgb(51,51,51);">You must pick one of the following values:&nbsp;</span><ul><li><span style="color: rgb(61,60,64);"><code>&quot;arrow&quot;</code>: The edge is represented by an arrow. This is the default setting.&nbsp;</span></li><li><span style="color: rgb(61,60,64);"><code>&quot;box&quot;</code>: The edge is represented by a box of a head asset containing a box of a tail asset.</span></li><li><span style="color: rgb(61,60,64);"><code>&quot;reversed-box&quot;</code>: The edge is presented as a box of a tail asset containing the box of a head asset.</span>&nbsp;</li></ul></li></ul><pre>&nbsp;</pre><pre>&quot;label&quot;: &quot;Groups&quot;</pre><ul><li><span style="color: rgb(61,60,64);"><span style="color: rgb(51,51,51);">Determines the name that is displayed on edges in the diagram.</span> </span></li><li><span style="color: rgb(61,60,64);"> <span style="color: rgb(79,79,79);">This setting is optional.&nbsp;</span> </span></li><li><span style="color: rgb(61,60,64);"> <span style="color: rgb(79,79,79);">If you do not specify a label, the role or co-role of the relation type from the operating model is used, in both the diagram view and the result diagram. <br />If <code>&quot;roleDirection&quot;</code> is <code>true</code>, DGC uses the role, if it is <code>false</code>, DGC uses the co-role.<br /><br /></span></span></li></ul><h2><span style="color: rgb(61,60,64);">Creating correct diagram views</span></h2><p><span style="line-height: 1.4285715;"> <br /> </span></p><p>JSON is a case-sensitive language. This means that you have to use the exact name/value pairs that we described here.&nbsp;</p><p>The JSON view needs to contain a Diagram section, a nodes section and an edges section. The order of the sections is irrelevant. Layout to improve readability (spaces, tabs, empty lines), is irrelevant.&nbsp;</p><p><span style="line-height: 1.4285715;">DGC checks some aspects of the syntax of your JSON view constantly as you are typing.&nbsp;<br /> </span> <span style="line-height: 1.4285715;">If the JSON view is incorrect, DGC will signal this immediately and will not allow you to save the diagram view.&nbsp;<br /> </span> <span style="line-height: 1.4285715;">For example, if you omit the required comma between two name/value pairs, DGC will show an error message, and highlight the offending line.&nbsp;<br /> </span> <span style="line-height: 1.4285715;">Also, if you make a typo in a value, DGC will show an error, listing out all the allowed values.&nbsp;</span></p><p>DGC will perform some other syntax and semantic checks when you press the Save button.&nbsp;<br /> <span style="line-height: 1.4285715;">For example, as we pointed out earlier, the resulting graph needs to be connected: all nodes need to be reacheable from all other nodes.&nbsp;</span></p><p><span style="line-height: 1.4285715;">You can also create other name/value pairs, with a name that is not listed here. These name/value pairs are allowed but will be ignored.&nbsp;<br />You can use this to your advantage, for example to add comments to the JSON view, but be aware that if you make a typo in the name, DGC will not signal this, instead it will ignore that line.&nbsp;&nbsp;</span></p><p>&nbsp;</p><p>Note: When you click the save button, DGC performs various checks:</p><ol><li>Whether each resource ID (conceptTypeId or binaryFactTypeId) that you specified exists in the operating model.&nbsp;<br />If a resource ID does not exist, it is removed from the diagram view and a warning and an error are displayed. DGC does not save your view unless you correct the errors.<br /><ac:image><ri:attachment ri:filename="error-message-wrongid.png" /></ac:image><br /><ac:image><ri:attachment ri:filename="error-example-wrongid.png" /></ac:image></li><li>Whether the diagram view is a connected graph.&nbsp;<br />A diagram view is a connected graph when every node can be reached from any other node by traversing the edges. If the diagram is not connected, a warning is displayed.&nbsp;<br />DGC does not save your view unless you correct the errors.<br /><ac:image><ri:attachment ri:filename="error-message-incomplete-diagram.png" /></ac:image><br /><br /></li></ol><p>&nbsp;</p>
