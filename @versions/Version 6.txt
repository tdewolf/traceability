<p>Location:&nbsp;<strong>Asset page &gt; Traceability</strong></p><p>In the Data Governance Center, you can display an asset's traceability.&nbsp;Traceability is essential for impact analysis.&nbsp;This feature helps you to quickly see to which other assets a specific asset is&nbsp;connected. Two assets are connected if you can navigate from one asset to another by traversing one or more relations.</p><p>With this feature, you can keep track of:</p><ul style="list-style-type: square;"><li>Relationships between assets, even if they are indirect</li><li>People involved with a specific asset</li><li>References in other assets</li></ul><p>Example: You can check the governance resources to which an asset is connected. You can find in which database tables a certain business term is stored.</p><p>Not every relation that is displayed in an asset's <strong>Overview</strong> tab is taken into account for the traceability.</p><ul><li>The <strong>Overview</strong> tab displays each relation type that is defined in the attribute assignment group of the asset type and the first relation type in a trace.</li><li>The <strong>Traceability</strong> tab displays each relation type that is defined in the traceability assignment group of the asset type.</li></ul><p>&nbsp;</p>
